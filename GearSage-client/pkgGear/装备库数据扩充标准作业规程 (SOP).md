# 装备库数据扩充标准作业规程 (SOP)

**版本**: 1.0
**日期**: 2026-03-27
**目标**: 建立一套标准化、可执行、可迭代的流程，用于高效、准确、无重复地向装备库中添加新的装备数据。

---

## 核心原则

1.  **防错而非纠错**: 在数据录入源头建立严格的校验机制，避免脏数据入库。
2.  **配置化而非硬编码**: 数据源、校验规则等应通过配置文件管理，便于维护和扩展。
3.  **合规与尊重**: 严格遵守网站服务条款，优先采用官方API或人工辅助方式，规避法律风险。
4.  **人机协作**: 将人类从重复性工作中解放出来，专注于信息核对；将繁琐的格式化与校验工作交给自动化脚本。

---

## 问题与解决方案总览

| 核心问题 | 解决方案 |
| :--- | :--- |
| **数据源分散，口径不一** | 建立并维护 `source_config.json` 配置文件，为每个品牌和品类预设权威的数据源入口。 |
| **数据重复录入** | 在录入前，强制执行基于“品牌+型号+年份”的唯一性检查流程。 |
| **数据准确性难以保证** | 1. 遵循数据源“信任链”；2. 采用标准化的Excel模板；3. 执行 `pre-check.js` 预校验脚本。 |
| **反爬虫与法律合规风险** | 明确“人机协作”为首选模式：机器人负责定位URL和后续处理，人类负责核心信息的复制粘贴，完全避免自动爬取行为。 |

---

## 标准作业流程 (SOP)

整个流程分为三个核心阶段：**准备与标准化**、**数据采集与录入**、**校验与导入**。

### **第一阶段：准备与标准化 (一次性设置)**

此阶段为所有后续工作奠定基础，通常只需在添加新品牌或新品类时执行。

1.  **维护数据源配置文件**:
    *   **文件**: `/Users/tommy/GearSage/GearSage-client/pkgGear/source_config.json` (如果不存在，请创建此文件)。
    *   **操作**: 当需要添加一个新品牌（如 "Evergreen"）时，向此文件添加新的条目。
    *   **示例**:
        ```json
        {
          "daiwa": {
            "brand_id": 2,
            "sources": [
              {
                "type": "rod",
                "entry_point": "https://www.daiwa.com/jp/fishing/item/rod/index.html",
                "priority": 1,
                "source_type": "official_site"
              }
            ]
          },
          "evergreen": {
            "brand_id": 15, // 假设在 brand.xlsx 中为 Evergreen 分配的 ID 是 15
            "sources": [
              {
                "type": "rod",
                "entry_point": "https://www.evergreen-fishing.com/products/rod/",
                "priority": 1,
                "source_type": "official_site"
              }
            ]
          }
        }
        ```

2.  **维护Excel录入模板**:
    *   **位置**: `/Users/tommy/GearSage/GearSage-client/rate/excel/_templates/` (建议创建此目录)。
    *   **操作**: 为 `brand.xlsx`, `reel.xlsx`, `rod_detail.xlsx` 等核心文件创建带有详细说明的模板副本。模板应冻结首行，并为每个字段提供格式说明和示例值。

### **第二阶段：数据采集与录入 (每次添加时执行)**

这是添加新装备时的核心执行步骤。以添加一款新的 **Daiwa 鱼竿** 为例。

1.  **步骤 1: 任务启动与查重**
    *   **操作者**: 您
    *   **指令**: “准备添加新的 Daiwa 鱼竿: 'HEARTLAND 802MFS-SV AGS23'”。
    *   **机器人/我**:
        1.  读取 `brand.xlsx`，确认 "Daiwa" 的 `brand_id` 为 `2`。
        2.  读取 `rod.xlsx`，筛选 `brand_id` 为 `2`，`model` 包含 "HEARTLAND"，且 `model_year` 为 `2023` 的所有行。
        3.  **报告结果**:
            *   **若找到匹配项**: “报告：该型号已存在。请确认是否要为其补充新的规格数据。如果是，请直接操作 `rod_detail.xlsx`。” -> **流程终止**。
            *   **若未找到**: “确认：此为新型号，可以录入。正在为您查找数据源...” -> **进入下一步**。

2.  **步骤 2: 定位数据源 (人机协作)**
    *   **机器人/我**:
        1.  读取 `source_config.json`，找到 `daiwa` -> `rod` 的 `entry_point`。
        2.  （模拟人类导航）从入口页面找到 'HEARTLAND 802MFS-SV AGS23' 的最终产品规格页面。
        3.  **报告结果**: “报告：已定位到官方产品页面: [https://www.daiwa.com/jp/.../heartland_ags23.html]。为确保合规，请您访问此链接，手动复制页面中的规格参数。”

3.  **步骤 3: 数据录入**
    *   **操作者**: 您
    *   **操作**:
        1.  打开上述链接，将主数据（型号、年份、图片链接等）和规格数据（长度、重量、饵重范围等）复制出来。
        2.  打开 `_templates` 中的 `rod.xlsx` 和 `rod_detail.xlsx` 模板。
        3.  将复制的信息，按照模板规范，填写到两个文件的新行中。确保为新行分配一个唯一的 `id` 和 `variantId`。

### **第三阶段：校验与导入 (自动化)**

1.  **步骤 1: 执行预校验**
    *   **操作者**: 您
    *   **指令**: “执行数据预校验”。
    *   **机器人/我**:
        1.  运行 `pre-check.js` 脚本 (此脚本需要被创建)。
        2.  脚本自动对 `/rate/excel/` 目录下的所有 Excel 文件执行以下检查：
            *   **完整性**: 必填项（如 `id`, `brand_id`, `model`）是否为空？
            *   **唯一性**: `id` 在各自文件中是否唯一？主数据中的 `brand_id` + `model` + `model_year` 组合是否唯一？
            *   **关联性**: `*_detail.xlsx` 中的外键 `*_id` 是否能在对应的主数据文件中找到？
            *   **格式**: `id` 是否为纯数字？`images` 是否用 `,` 分隔？
        3.  **报告结果**:
            *   **若有错误**: “报告：发现3个问题。1. `rod_detail.xlsx` 第 112 行: `rod_id` '999' 无法在 `rod.xlsx` 中找到。 2. ...” -> **流程暂停，返回第二阶段第3步进行修正**。
            *   **若无错误**: “报告：预校验通过，所有数据格式正确、关联无误。” -> **进入下一步**。

2.  **步骤 2: 执行最终导入**
    *   **操作者**: 您
    *   **指令**: “确认导入数据”。
    *   **机器人/我**:
        1.  执行现有的 `import_gear_excel.js` 脚本。
        2.  **报告结果**: “报告：数据已成功导入数据库，并已更新客户端筛选文件 `Data.js`。” -> **流程结束**。

## 迭代与扩展

本规程是可扩展的：

*   **添加新品牌**: 只需在 `brand.xlsx` 中添加新行，并在 `source_config.json` 中配置其数据源。
*   **增加校验规则**: 如果发现新的常见错误，只需在 `pre-check.js` 脚本中增加新的校验函数即可。
*   **支持新品类**: 例如未来要添加“鱼线”，则需要创建 `line.xlsx` 和 `line_detail.xlsx` 的模板，并在 `source_config.json` 和 `pre-check.js` 中添加对 `line` 类型的支持。
