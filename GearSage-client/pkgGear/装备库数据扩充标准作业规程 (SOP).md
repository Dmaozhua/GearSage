# 装备库数据扩充标准作业规程 (SOP)

**版本**: 1.1
**日期**: 2026-03-27
**变更日志**:
*   **v1.1**: 引入 `Scrapling` 库和本地化脚本执行方案，将数据采集阶段从“手动复制”升级为“半自动化执行”，大幅提升效率和准确性。确立了 Python 虚拟环境的最佳实践。

**目标**: 建立一套标准化、可执行、可迭代的流程，用于高效、准确、无重复地向装备库中添加新的装备数据。

---

## 核心原则

1.  **防错而非纠错**: 在数据录入源头建立严格的校验机制，避免脏数据入库。
2.  **配置化而非硬编码**: 数据源、校验规则等应通过配置文件管理，便于维护和扩展。
3.  **合规与尊重**: **人机协作，本地执行**。由机器人生成采集脚本，由用户在本地环境执行，完全规避云端爬取带来的法律风险。
4.  **人机协作**: 机器人负责编写和维护采集/校验脚本，人类负责执行和最终确认，实现效率最大化。

---

## 问题与解决方案总览

| 核心问题 | 解决方案 |
| :--- | :--- |
| **数据源分散，口径不一** | 建立并维护 `source_config.json` 配置文件，为每个品牌和品类预设权威的数据源入口。 |
| **数据重复录入** | 在录入前，强制执行基于“品牌+型号+年份”的唯一性检查流程。 |
| **数据准确性难以保证** | 1. 遵循数据源“信任链”；2. **由自动化脚本直接生成结构化数据**；3. 执行 `pre-check.js` 预校验脚本。 |
| **反爬虫与法律合规风险** | **采纳“我写脚本，您执行”的人机协作模式**。利用 `Scrapling` 库在本地环境执行数据采集，脚本由机器人提供，规避了云端直接爬取的风险。 |

---

## 标准作业流程 (SOP)

整个流程分为三个核心阶段：**准备与标准化**、**数据采集与录入**、**校验与导入**。

### **第一阶段：准备与标准化 (一次性或低频操作)**

此阶段为所有后续工作奠定基础。

1.  **搭建本地 Python 环境 (首次执行)**
    *   **目标**: 确保执行环境统一。
    *   **步骤**:
        1.  安装 `Homebrew` (若无)。
        2.  执行 `brew install python` 安装最新 Python。
        3.  在项目根目录创建虚拟环境: `python3 -m venv venv`。
        4.  使用虚拟环境的 pip 安装所需库: `./venv/bin/pip install scrapling`。

2.  **维护数据源配置文件**:
    *   **文件**: `/Users/tommy/GearSage/GearSage-client/pkgGear/source_config.json`。
    *   **操作**: 当需要添加新品牌时，向此文件添加新的数据源入口。

3.  **维护Excel录入模板**:
    *   **位置**: `/Users/tommy/GearSage/GearSage-client/rate/excel/_templates/`。
    *   **说明**: 虽然采集过程自动化，但模板仍作为数据结构的最终参照和手动小规模补充的备用方案。

### **第二阶段：数据采集与录入 (半自动化)**

这是添加新装备时的核心执行步骤。以采集 **Daiwa 纺车轮** 数据为例。

1.  **步骤 1: 任务启动与查重**
    *   **操作者**: 您
    *   **指令**: “帮我采集 Daiwa 官网所有纺车轮的数据”。
    *   **机器人/我**:
        1.  执行查重逻辑，确认哪些是需要新增的型号。
        2.  **报告结果**: “确认：官网共有25个型号，数据库中已有20个，将为您采集剩余5个新型号的数据。正在编写采集脚本...”

2.  **步骤 2: 编写并提供采集脚本**
    *   **机器人/我**:
        1.  根据 `source_config.json` 中的入口，基于 `Scrapling` 的功能，编写一个定制化的 `daiwa_reel_scraper.py` 脚本。
        2.  将脚本内容和执行命令提供给您。

3.  **步骤 3: 本地执行采集**
    *   **操作者**: 您
    *   **操作**:
        1.  将脚本保存到本地。
        2.  在项目根目录打开终端，执行命令: `./venv/bin/python daiwa_reel_scraper.py`。
        3.  脚本会自动运行，并将采集到的数据保存为结构化的 `daiwa_reels_raw.csv` 文件。

4.  **步骤 4: 提交采集结果**
    *   **操作者**: 您
    *   **操作**: 将生成的 `daiwa_reels_raw.csv` 文件提供给我，用于后续处理。

### **第三阶段：校验与导入 (自动化)**

1.  **步骤 1: 数据转换与预校验**
    *   **机器人/我**:
        1.  读取您提供的 `.csv` 文件。
        2.  将其内容转换为符合 `reel.xlsx` 和 `spinning_reel_detail.xlsx` 格式的数据。
        3.  运行 `pre-check.js` 脚本进行最终校验（关联性、唯一性等）。
        4.  **报告结果**:
            *   **若有错误**: “报告：数据在转换或校验中发现问题... 将尝试自动修复或请您确认。”
            *   **若无错误**: “报告：数据校验通过，已生成待导入的 Excel 文件。”

2.  **步骤 2: 确认并执行最终导入**
    *   **操作者**: 您
    *   **指令**: “确认导入”。
    *   **机器人/我**:
        1.  执行 `import_gear_excel.js` 脚本。
        2.  **报告结果**: “报告：数据已成功导入数据库。” -> **流程结束**。

## 迭代与扩展

本规程是可扩展的：

*   **添加新数据源**: 只需在 `source_config.json` 中配置新入口，并由我为您编写一个新的采集脚本。
*   **增加校验规则**: 只需在 `pre-check.js` 脚本中增加新的校验函数。
*   **支持新品类**: 创建新品类的 Excel 模板，并编写对应的采集和转换逻辑。
