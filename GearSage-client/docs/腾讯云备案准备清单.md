# 腾讯云备案准备清单

版本：v1.0  
状态：当前执行版  
适用对象：Tommy / Codex / 备案准备

---

## 一、目的

这份清单用于准备 GearSage 中国内地正式环境的 ICP 备案材料。

当前口径：

- 本地开发不依赖备案
- 正式内地域名要上线并稳定对外提供服务，备案仍是前置事项
- 本清单只整理“准备材料与填写口径”，不替代腾讯云备案流程说明

参考官方文档：

- [腾讯云 ICP 备案简介](https://cloud.tencent.com/document/product/243/50003)

---

## 二、当前适用范围

用于确认以下信息：

- 正式环境是否继续放在中国内地
- 哪个主体负责备案
- 哪些域名进入备案
- 后续微信审核、短信、静态资源是否复用同一主体

当前建议：

- 如果正式 API、静态资源继续在中国内地提供服务，就按个体工商户主体准备备案

---

## 三、备案前需要先明确的事项

### 1. 主体

需要确认：

- 主体类型：个体工商户
- 主体名称
- 统一社会信用代码
- 经营者姓名
- 经营者身份证

### 2. 域名

需要确认：

- 备案使用的主域名
- API 域名
- 静态资源域名
- 域名注册人是否与备案主体一致

### 3. 服务器

需要确认：

- 使用腾讯云中国内地服务器
- 对应实例地域
- 对应实例可用于备案接入

---

## 四、材料清单

### 主体材料

- 营业执照
- 经营者身份证正反面
- 经营者手机号
- 联系邮箱

### 域名材料

- 域名证书或注册信息截图
- 域名所有人信息
- 域名解析关系说明

### 服务说明材料

- 产品名称：GearSage
- 服务简介
- 网站/小程序主要功能说明
- 涉及用户生成内容与审核说明

---

## 五、腾讯云备案填写口径建议

建议统一按下面的方向描述，不要写成泛泛“论坛”：

- 服务名称：GearSage
- 服务内容：路亚钓鱼装备经验、提问、鱼获与钓行内容分享
- 主要用户：路亚钓鱼用户
- 内容治理：平台对发帖、评论、资料和图片提供审核与人工复核机制

这样更符合当前产品真实形态，也能和微信审核资料口径保持一致。

---

## 六、和后续资料包的关系

备案准备完成后，可直接复用到：

- 微信审核主体说明
- 腾讯云短信实名资质
- 安全评估准备包中的主体信息

---

## 七、当前执行建议

1. 先固定备案主体和域名归属
2. 再整理主体与联系方式材料
3. 再进入腾讯云备案流程填写
4. 备案过程中同步沉淀截图和材料，后续微信审核可复用

---

## 八、完成标准

满足以下条件即可视为准备完成：

- 备案主体已确认
- 域名归属已确认
- 营业执照与身份证材料齐全
- API / 静态域名与主体关系已能自洽说明
- 已具备在腾讯云控制台发起备案的材料条件
