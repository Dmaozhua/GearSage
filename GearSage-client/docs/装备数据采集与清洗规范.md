# GearSage 装备数据采集与清洗规范 (v1.0)

## 核心原则 (Core Principles)
- **统一数据出口**：无论是哪个品牌（Daiwa, Shimano, Megabass），最终生成的 Excel 表头必须严格遵守 GearSage 的基础导入模板结构。
- **语义字段融合**：不同品牌官网对同一属性的叫法不同（如 buoyancy vs sinkingspeed），在脚本层必须将其映射为统一的 GearSage 标准字段，绝不允许在最终表里出现意思相同但名字不同的列。
- **扩展字段后置**：对于基础模板未能涵盖的品类特有字段（如假饵的潜水深度、鱼竿的含碳量等），统一作为扩展字段追加在标准表头的末尾。

---

## 1. 字段映射与融合规范 (Field Mapping & Fusion Standards)

### 1.1 假饵 (Lure) 核心融合规则
在处理 Lure 变体数据时，统一执行以下映射：
- **沉水属性 (Buoyancy)**：统一映射至 `sinkingspeed`。禁止保留多余的 `buoyancy` 字段。
- **价格 (Price)**：统一映射至 `referenceprice`。
- **重量 (Weight)**：统一映射至 `WEIGHT`。
- **长度/尺寸 (Length/Size)**：统一映射至 `length`（如 120mm）。`size` 字段留空或视情况作他用。
- **官方编码 (Product Code/JAN)**：统一映射至 `AdminCode`。
- **颜色 (Color)**：不管抓取时叫 `color` 还是 `color_name`，一律映射为 `COLOR`。删除无用的 `color_code` 占位符。
- **包装数量 (Quantity)**：软饵特有。不管官网叫 `quantity` 还是 `入数`，一律合并输出为扩展字段 `quantity (入数)`，专供软饵表使用。
- **鱼钩尺寸 (Hook)**：统一合并为 `hook_size`。

### 1.2 渔轮 (Reel) 核心融合规则
- **速比 (Gear Ratio)**：统一映射至 `gear_ratio`，并需衍生出标准化的 `gear_ratio_normalized`（如 P, H, XG）。
- **刹车力/最大阻力 (Drag)**：统一提取数值至 `max_drag`。
- **重量 (Weight)**：统一映射至 `weight`。
- **轴承数 (Bearings)**：统一提取为 `bearings`。

### 1.3 鱼竿 (Rod) 核心融合规则
- **节数 (Pieces)**：统一映射至 `pieces`。
- **收纳长度 (Closed Length)**：统一映射至 `closed_length`。
- **先径/元径 (Tip/Butt Dia)**：分别映射至 `tip_diameter` 和 `butt_diameter`。

## 2. 唯一ID生成与分配规范 (ID Allocation & Prefix Standards)
为了防止不同品牌的同类型装备在数据合并导入时发生主键冲突（如均以 1000 开始递增），强制执行以下字符串前缀 ID 规则：

**前缀结构：[品牌首字母] + [品类首字母] + [层级] + [自增数字]**
- **品牌 (Brand)**: Daiwa=`D`, Shimano=`S`, Megabass=`M`
- **品类 (Category)**: Rod=`R`, Reel=`RE`, Lure=`L`, Line=`LN`
- **层级 (Level)**: Main/Master = 无后缀, Detail/Variant = 追加 `D`

**具体分配表**：
| 品类 & 品牌 | 主表前缀 (Main ID) | 子表前缀 (Detail ID) | 起始数字范围 |
|---|---|---|---|
| **Lure (假饵)** | `DL`, `SL`, `ML` | `DLD`, `SLD`, `MLD` | `1000` (Main), `10000` (Detail) |
| **Rod (鱼竿)** | `DR`, `SR`, `MR` | `DRD`, `SRD`, `MRD` | `1000` (Main), `10000` (Detail) |
| **Line (鱼线)** | `DLN`, `SLN`, `MLN` | `DLND`, `SLND`, `MLND` | `1000` (Main), `10000` (Detail) |
| **Reel (纺车轮)** | `DRE`, `SRE`, `MRE` | `DRED`, `SRED`, `MRED` | `1000` (Main), `10000` (Detail) |
| **Reel (水滴轮)** | `DRE`, `SRE`, `MRE` | `DRED`, `SRED`, `MRED` | `5000` (Main), `50000` (Detail) |

*注意*：在 JS 清洗脚本（如 `to_excel_megabass_lure.js`）中生成 ID 时，必须通过拼接实现，例如 `` const currentLureId = `ML${lureIdCounter++}`; ``。

## 3. 导出 Excel 表头排序规范 (Excel Export Header Ordering)
**强制要求**：在使用 `xlsx` 库导出 JSON 为 Excel 时，必须显式传入 `header` 配置，以保证前 N 列与基础模板完全一致。

**示例（Lure）**：
```javascript
const wsHardbait = xlsx.utils.json_to_sheet(hardbaitDetailRows, { 
    header: ["id", "lure_id", "SKU", "WEIGHT", "length", "size", "sinkingspeed", "referenceprice", "created_at", "updated_at"] 
});
```
*作用*：保证上述标准字段永远在最前面，其他特有字段（如 `COLOR`, `AdminCode`, `hook_size`, `depth`）会自动按顺序追加在右侧，确保导入脚本的完全兼容性。

## 4. 主图图片下载与映射规范 (Image Downloading & Path Mapping)
为应对防爬与资源加载问题，图片处理统一走本地化流程：
1. **下载机制**：使用 Python 脚本（如 `download_images_lure.py`）结合 `ThreadPoolExecutor` 多线程并发下载，取消单线程的休眠限速以提升效率。
2. **防爬突破**：对于 Shimano 等有严格防护的站点，使用 `curl_cffi` 模拟浏览器指纹（`impersonate="chrome"`）进行抓取和下载。
3. **路径映射**：
   - 图片统一下载到 `GearSage-client/pkgGear/images/<brand>_<category>/` 目录下。
   - 命名规则优先使用 `<SKU/Model>_main.jpg`。
   - 在 JS 处理脚本中，优先读取本地相对路径存入 `images` 字段；若本地图片缺失，再回退使用原始 `main_image_url` 远程链接。
4. **容错处理**：必须包含 `os.makedirs(exist_ok=True)` 和 `try-except` 块，防止因权限或网络超时导致整个批处理中断。

## 5. 关键数据提取策略 (Key Data Extraction Strategies)
- **Model Year (上市年份)**：
  - **Shimano**：官网无独立字段，需通过正则 `r'20[0-2]\d\s*年'` 从商品描述段落（`.product__description_section`）中提取。
  - **Daiwa**：通常可从 SKU 变体名的前两位提取（如 `22EXIST` -> 2022），使用正则 `r"^(\d{2})(?!\d)\s*(.*)"`（利用负向先行断言避免误伤 `4000D` 等型号数字）。
- **Submodel/Variant 描述**：
  - 处理复杂的变体特性时（如 Megabass 的 `subname`、Daiwa 的 `series_description` 和 `variant_description`），尽量保留在扩展列中，以辅助后续运营人工审阅。

---

## 5. 执行工作流 (Workflow Checklist)
新增任何品牌的抓取与导出时，需严格执行以下工业化流水线：
1. **抓取阶段** (`scrapers/<brand>/<category>.py`)：抓取全量数据，保存为 `data_raw/<brand>_<category>.json`。
2. **图片下载** (`download_images_<category>.py`)：读取 JSON，并发下载主图至 `images/` 目录，更新 JSON 本地路径。
3. **清洗与映射** (`to_excel_<brand>_<category>.js`)：执行字段语义融合（第1节），应用表头强制排序（第2节），生成 `_import.xlsx` 最终入库文件。
4. **核对审查**：打开生成的 Excel，确认前 N 列是否与原表对齐，确认冗余同义字段（如 `buoyancy`）是否已被清理干净。
