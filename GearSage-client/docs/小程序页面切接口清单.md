## 一、P0 接口基线

### 1. 鉴权

- `POST /auth/send-code`
- `POST /auth/login`
- `POST /auth/refresh`
- `POST /auth/logout`
- `GET /auth/me`

### 2. 用户

- `GET /mini/user/info?id=:id`
- `POST /mini/user/update`
- `GET /mini/user/points`

### 3. 评论

- `GET /mini/comment?topicId=:id`
- `PUT /mini/comment`
- `POST /mini/comment/like`
- `DELETE /mini/comment?commentId=:id`

### 4. 上传

- `POST /upload`
- `POST /upload/image`
- `POST /upload/avatar`
- `POST /upload/background`

### 5. 标签 / 虚拟权益（P1 已开始）

- `GET /mini/tag/usable`
- `GET /mini/tag/used`
- `POST /mini/tag/used`
- `GET /mini/tag/points`
- `POST /mini/tag/redeem`
- `GET /mini/goods?type=0`
- `POST /mini/goods`

## 四、P0 联调结论

- 鉴权第一版：已形成 `send-code -> login -> me -> refresh -> logout` 闭环
- 用户第一版：当前用户资料读取、编辑资料保存、积分读取已接到独立后台
- 评论第一版：评论列表、发表评论、删除评论、评论点赞已切到独立后台
- 上传第一版：帖子图片、头像、背景图上传已切到独立后台本地上传目录
- 标签第一版：`my-tags / tag-manager` 已切到独立后台
- 虚拟权益商城第一版：`points-exchange` 的 `type=0` 页签已切到独立后台

- `GET /mini/user/info` 虽已在 API 仓库实现，但目前没有主页面正式消费
- 任务接口 `/mini/taskFeat*` 当前仍为占位实现，尚未形成真实任务业务链路
- 仍存在少量非 P0 模块的云开发残留，主要集中在 `pkgGear`
