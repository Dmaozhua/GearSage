## **MySQL 8 建库脚本（DDL，可直接执行）**

说明：
- `_id` 一律用 `CHAR(32)`（CloudBase docId 常见 32 hex；如你们不是 32，也可改 `VARCHAR(64)`）
- 数组/对象字段用 `JSON`
- 时间用 `DATETIME(3)`（对齐 JS Date 精度）
- 兼容字段不在 MySQL 里重复保留，统一落到标准字段；如要追溯原始文档，可额外加 `raw_doc JSON`（这里不强制）

  union_id VARCHAR(64) NULL,
  session_key VARCHAR(128) NULL,

  nick_name VARCHAR(64) NOT NULL,
  avatar_url VARCHAR(512) NOT NULL,
  bio VARCHAR(512) NOT NULL DEFAULT '',
  background VARCHAR(512) NOT NULL DEFAULT '',
  ship_address VARCHAR(512) NOT NULL DEFAULT '',

  status TINYINT NOT NULL DEFAULT 0,
  points INT NOT NULL DEFAULT 0,
  level INT NOT NULL DEFAULT 1,

  total_login_days INT NOT NULL DEFAULT 1,
  continuous_login_days INT NOT NULL DEFAULT 1,
  last_auth_time DATETIME(3) NULL,

  invite_code VARCHAR(32) NULL,
  invited_by_user_id CHAR(32) NULL,
  invite_code_used VARCHAR(32) NULL,

## **“完整数据库”交付 = Schema + 数据迁移导入方案**

### A) 从云开发导出数据
你们已有脚本使用 `@cloudbase/node-sdk`（例如清理任务记录 [clear-task-records.js](file:///d:/Xiaochengxu/Diaoyoushuo/scripts/clear-task-records.js#L1-L69)、Excel 导入 [import_excel_to_db.js](file:///d:/Xiaochengxu/Diaoyoushuo/scripts/import_excel_to_db.js#L158-L232)），建议导出同样走 node-sdk：
- 对每个集合 `db.collection(name).skip(page*1000).limit(1000).get()` 分页拉取，输出 NDJSON（每行一个 doc）
- 导出集合清单按上面 “已识别集合” 执行；其中 `bz_tag/bz_user_tag` 只在你们要保留历史兼容时导出

### B) 导入到 MySQL（关键映射规则）
- `_id` → `id`（MySQL 主键）
- 时间字段：CloudBase Date 直接转 `DATETIME(3)`
- topics 的数组/对象：直接写入 JSON 列（`images/environments/ratings/...`）
- 评论：把原 payload（除 topicId/userId/replayCommentId/isVisible/createTime/updateTime 外）整体放 `payload_json`
- task_day：按照你们云函数的“凌晨 2 点刷新窗口”规则计算（见 [miniApi/index.js](file:///d:/Xiaochengxu/Diaoyoushuo/cloudfunctions/miniApi/index.js#L370-L385)），推荐导入时就落库：
  - `task_day = DATE(created_at - INTERVAL 2 HOUR)`（以服务器时区为准；如果你们用 UTC，需要先统一时区）

### C) 兼容与清理建议（避免脏数据）
- `bz_points_goods.tagId/tag_id` 可能出现 Number 精度丢失（你们已在 [fixUserTags/index.js](file:///d:/Xiaochengxu/Diaoyoushuo/cloudfunctions/fixUserTags/index.js#L1-L70) 处理过类似问题）：迁移时一律当字符串处理
- `bz_user_tags` 里可能出现嵌套 `data` 文档（脚本里也处理过，见 [grant_all_tags_to_user.js](file:///d:/Xiaochengxu/Diaoyoushuo/scripts/grant_all_tags_to_user.js#L86-L115)）：导入前先“解包”成扁平结构
- 用户 id 兼容：部分旧数据可能把 `openId` 写进 `topic.userId`（你们代码里已兼容读取，见 [miniApi/index.js](file:///d:/Xiaochengxu/Diaoyoushuo/cloudfunctions/miniApi/index.js#L2049-L2063)）：迁移时建议修正为真正 `users.id`
