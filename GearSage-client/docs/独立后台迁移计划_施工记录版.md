# 十、迁移执行总原则

## 10.1 先做后端骨架，再做前端替换
原则：

1. 先把独立后台核心接口做出来
2. 再把前端从云开发调用切到后台接口
3. 最后清理云函数和云数据库直查

## 10.2 不做大爆炸式迁移
当前项目采用分模块渐进迁移：

- 先帖子 / 评论 / 上传 / 登录 / 用户
- 再标签 / 商城 / 任务
- 再装备库
- 最后清理历史云开发

## 10.3 每做完一步，必须回写本文档
任何新增内容都必须同步更新：

- 已完成接口
- 已完成数据库表
- 已完成服务器变更
- 当前运行方式
- 下一步计划

# 十四、当前已知问题与经验记录

## 14.1 终端 heredoc 容易被打乱
当前施工过程中已经多次出现：

- `cat > file <<'EOF'` 输入中混入其他命令
- 返回结果和下一条命令串在一起
- `EOF` 未单独结束

### 处理原则
以后写多行文件时必须遵守：

1. 从 `cat > xxx <<'EOF'` 开始整段复制
2. 中途不插其他命令
3. 最后一行必须是单独的 `EOF`

## 14.2 上传路径曾判断错误
曾短暂把上传目录设为：

```text
/lhcos-data/data/gearsage
```

后发现会导致对象路径多一层 `/data`，因此正式修正为：

```text
/lhcos-data/gearsage
```

这是当前已拍板结果，后续不得回退。

## 14.3 `http://127.0.0.1/...` 会被 301
由于 Nginx 已将 80 端口统一跳转到 HTTPS，本机测试接口时：

- 要绕过 Nginx，直接打 `http://127.0.0.1:3000/...`
- 要测正式链路，打 `https://api.gearsage.club/...`
