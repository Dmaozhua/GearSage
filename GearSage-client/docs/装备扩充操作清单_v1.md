# 装备扩充操作清单 v1

## 目标

给后续装备扩充建立一套固定、可重复、低返工的操作方式。

当前约定：

*   `/Users/tommy/GearSage/GearSage-client/rate/excel/` 是最终基准
*   `/Users/tommy/GearSage/GearSage-client/pkgGear/data_raw/` 是抓取、清洗、导出中间层
*   默认冻结最终表结构，优先补内容

## 不变规则

1.  `brand.xlsx` 只扩充，不改既有品牌编号。
2.  装备主表、详情表统一使用字符串前缀主键。
3.  最终表字段名、字段顺序、sheet 名默认不动。
4.  新增字段前，先判断是不是可以通过补内容或修导出脚本解决。
5.  差异处理优先改导出脚本规则，不优先做重复人工修表。

## 最终表安全规则

下面这些规则是硬约束：

1.  **绝不允许整表覆盖**  
    `rate/excel` 只能按“当前品牌 + 当前品类 + 当前子类切片”精确替换，不能因为导入一批新数据，把同表里其他品牌或其他子类历史数据一起清掉。
2.  **绝不允许误删未触达数据**  
    如果这轮只更新 `Megabass reel`，那就只能替换 `MRE* / MRED*` 对应切片，不能动 `SRE*`、`DRE*`，也不能动 rod / lure / line。
3.  **绝不允许因前缀配置错误导致历史数据丢失**  
    每次新增品牌或新品类前，先确认主表 id 前缀、详情外键前缀、sync 脚本替换前缀三者一致；不一致时先修规则，不要直接同步。
4.  **绝不允许把有效旧值写成空值**  
    如果最终表已有字段值，而 import 行该字段为空，默认视为风险，不应在未确认的情况下把旧值冲掉。尤其要注意 `type`、图片路径、主外键、品牌号这类关键字段。
5.  **同步后必须先验，再允许进入导库**  
    同步到 `rate/excel` 后，先看差异报告、行数、前缀计数和关键字段抽样，确认没有误删误覆盖，再进入 `import:gear:check` 和正式导库。

## 每次扩充都按这个顺序做

1.  更新 `pkgGear/data_raw` 中的抓取结果或 `normalized.json`
2.  跑 `pre-check.js`
3.  跑对应 `to_excel_*` 导出脚本
4.  跑 `report_rate_excel_diffs.js`
5.  看差异报告
6.  只处理报告里的真实差异
7.  差异清零后，再更新 `rate/excel`
8.  运行导入前校验
9.  最后才进入导库和前后端联调

## 遇到差异时怎么判断

### 情况 1：新增型号没进最终表

说明抓取和导出有新增内容，但 `rate/excel` 还没吸收。

处理方式：

*   先确认新增内容正确
*   再手动更新最终表

### 情况 2：字段值落错列

说明导出规则不稳。

处理方式：

*   改 `to_excel_*` 脚本
*   重新导出
*   重新跑差异报告

### 情况 3：表头或 sheet 不一致

说明最终表规则和导出规则还没统一。

处理方式：

*   先定最终规则
*   再统一最终表和导出常量

### 情况 3.5：同步时可能误删历史数据

重点检查：

*   这轮替换是不是只命中了当前品牌 / 当前子类前缀
*   `before / incoming / replaced / after` 的数量关系是否合理
*   旧品牌、旧子类的数据行数是否保持不变
*   关键字段是否出现“原来有值，现在变空”

处理方式：

*   先停下，不要继续导库
*   先修 `sync_rate_excel_from_imports.js` 或导出脚本
*   修完后重新跑同步和差异报告

### 情况 4：想新增字段

只有在以下条件都满足时才建议加：

*   旧字段确实承载不了业务意义
*   该字段会长期使用
*   导出脚本、最终表、导入脚本、前后端都能一起改
*   对应文档会同步更新

## 完成标准

这轮扩充完成，至少应满足：

1.  `pre-check` 无 `error`
2.  `report_rate_excel_diffs.js` 报告清零或只剩已确认的差异
3.  `rate/excel` 中最终数据已确认
4.  `npm run import:gear:check` 通过
5.  导入脚本可消费
6.  若已执行正式导入，主包与装备库搜索索引已自动同步
7.  后端接口和前端页面能抽样验证
8.  同步到 `rate/excel` 的过程中，没有出现误删历史品牌、误覆盖其他子类、或关键字段被写空

## 一句话口径

**先补内容，再跑导出，再看差异，再修规则；不要先动最终表结构。**
