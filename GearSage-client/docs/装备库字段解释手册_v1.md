## 一、文档目的

这份文档用于解释当前装备库 Excel 中各类主表、详情表字段的含义。

重点不是数据库表结构，而是：

1. 每个字段在 Excel 维护层里表示什么
2. 它更偏“识别字段”“官方参数字段”“GearSage 推导字段”还是“补充字段”
3. 在产品设计时，哪些字段适合：
   - 列表页展示
   - 单一产品深读
   - 多款产品横向对比

说明：

- 本文档是**字段解释手册**，不是最终产品展示方案。
- 同名字段在不同品类中的意义可能不同，应按各自表说明理解。
- 部分字段是当前历史链路保留下来的“弱结构字段”或“兜底字段”，后续可再优化。

---

## 二、字段分层说明

为便于理解，本文档中把字段粗分为四类：

- `识别层`：回答“这是什么”
- `官方参数层`：厂家客观给出的结构化规格
- `GearSage 归纳层`：系统归一、解释、推导出的字段
- `补充层`：说明、备注、状态、额外信息

## 三、品牌主表 `brand.xlsx`

### 作用

品牌字典表，给所有装备主表的 `brand_id` 提供映射。

### 字段解释

| 字段 | 含义 | 层级 | 备注 |
|---|---|---|---|
| `id` | 品牌唯一编号 | 识别层 | 纯数字，后续只扩充不改号 |
| `name` | 品牌主名称 | 识别层 | 当前前后端最常用 |
| `name_en` | 英文名 | 识别层 | 可为空 |
| `description` | 品牌简介 | 补充层 | 品牌说明用 |
| `name_jp` | 日文名 | 识别层 | 可为空 |
| `name_zh` | 中文名 | 识别层 | 可为空 |
| `country` | 国家/地区 | 补充层 | 可为空 |
| `site_url` | 官网地址 | 补充层 | 品牌跳转用 |
| `created_at` | 创建时间 | 补充层 | 当前多为空 |
| `updated_at` | 更新时间 | 补充层 | 当前多为空 |

## 四、渔轮

## 4.1 主表 `reel.xlsx`

### 作用

渔轮系列主表。  
一行通常代表一个系列/主型号，不是单个具体 SKU。

### 字段解释

| 字段 | 含义 | 层级 | 备注 |
|---|---|---|---|
| `id` | 渔轮主型号 id | 识别层 | 例如 `SRE1000`、`DRE5000` |
| `brand_id` | 品牌 id | 识别层 | 关联 `brand.xlsx` |
| `is_show` | 是否在客户端展示 | 补充层/运行层 | `1` 展示且可查询，`0` 隐藏且不可查询 |
| `model` | 型号名 | 识别层 | 最核心的系列名称 |
| `model_cn` | 中文名 | 识别层 | 若没有则为空 |
| `model_year` | 年份 | 识别层 | 如 2025 |
| `alias` | 别名 | 识别层 | 用户搜索辅助 |
| `type_tips` | 类型提示 | GearSage 归纳层 | 当前渔轮里多为空，也可承接 BFS/泛用等提示 |
| `type` | 渔轮大类 | 识别层 | 当前口径为 `spinning` / `baitcasting` / `drum`；其中 `drum` 表示鼓轮 |
| `images` | 主图路径/主图 URL | 识别层 | 列表和详情顶部图片来源 |
| `created_at` | 创建时间 | 补充层 | 当前多为空或采集时间 |
| `updated_at` | 更新时间 | 补充层 | 当前多为空或采集时间 |
| `series_positioning` | 系列定位 | 补充层 | 系列一句话定位 |
| `main_selling_points` | 主要卖点 | 补充层 | 官方宣传点汇总 |
| `official_reference_price` | 官方参考价 | 官方参数层 | 系列价格区间 |
| `market_status` | 市场状态 | 补充层 | 在售/停产等 |

### 产品设计上值得特别关注的字段

如果从“单一产品细节了解 / 改装配件判断”的角度，这张表里最值得重点展示的是：

- `spool_diameter_mm`
- `spool_width_mm`
- `Nylon_lb_m` / `fluorocarbon_lb_m` / `pe_no_m`
- `handle_length_mm`
- `handle_knob_type`
- `handle_knob_exchange_size`

这些比单纯 `自重 / 速比 / 最大卸力` 更接近深玩家关心的问题。

### 4.4 当前 reel 施工中新增/待补字段口径

下面这些字段目前更偏“玩家补充字段”或“后续待补字段”。  
如果后续正式建列，建议按以下含义维护：

| 字段 | 含义 | 建议口径 |
|---|---|---|
| `handle_knob_material` | 握丸材质 | 如塑料 / 橡胶 / 木质 / 金属，优先保留原始说法 |
| `main_gear_material` | 大齿材质 | 优先保留官方或玩家资料里的原始材质表述 |
| `main_gear_size` | 大齿尺寸 | 只有资料明确时再填，不强行归一 |
| `minor_gear_material` | 小齿材质 | 深玩家补充字段 |
| `is_handle_double` | 是否双摇臂 | `1` 是，`0` 否 |
| `player_environment` | 玩家数据环境定位 | 来自玩家长期使用经验，不覆盖 `official_environment` |
| `player_positioning` | 玩家数据系列定位 | 玩家视角的一句话定位 |
| `player_selling_points` | 玩家数据主要卖点 | 来自玩家资料的卖点摘要，不直接冒充官方卖点 |

### `recommended_rig_pairing` 口径

这个字段承接“某个鱼竿子型号更适合搭配什么钓组或饵型”。

它不是饵重范围，也不是一句笼统的玩家定位。即使同属硬饵竿，也要尽量区分具体方向，例如 `Crankbait / Shad / Minnow / Spinnerbait / Chatterbait`，而不是只写 `硬餌`。

填写规则：

- 只写子型号维度的适配钓组或饵型。
- 多个项目按“最擅长 -> 合适”的顺序排列，重点放在最前面。
- 优先使用官网子型号说明、官方表格、官方技术页。
- 官网没有明确说明时，再用白名单辅助站补充判断。
- 不把来源说明写进字段值；来源证据应放在 sidecar evidence 或流程文档里。

示例：

- `Neko Rig / Down Shot / No Sinker / Small Rubber Jig`
- `Crankbait / Shad / Minnow / Spinnerbait`
- `Frog / Punching / Heavy Texas`
- `Eging / Tip-run Eging / Sinker Rig`
