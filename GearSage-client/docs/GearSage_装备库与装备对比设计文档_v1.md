# GearSage 装备库与装备对比设计文档 v1

版本：v1.0  
状态：可直接交给 Codex 进入实现拆分  
适用范围：GearSage-client / GearSage-api / Gear 数据导入链 / 后续文档回写  
更新时间：2026-04-02

---

## 一、文档目标

本文档用于在现有装备库第一版基础上，重新定义 GearSage 装备库的产品结构、详情页信息分层、对比页设计、对比字段体系、不同装备类型的对比深度，以及与求推荐的联动关系。

本文档重点解决以下问题：

1. 当前装备库 UI 还缺什么
2. 不同装备类型在详情与对比时，哪些信息应该被强化
3. 不同装备类型的对比“深度”为什么不能一样
4. 什么信息属于官网参数，什么信息属于 GearSage 人工归纳
5. 在“用户已经知道自己要对比同类同用途装备”的前提下，对比页应该怎么收敛信息
6. 在“用户其实选错了比较对象”的情况下，如何给出克制但有效的提醒
7. 如何让装备库继续保持可扩充，而不是做成一次性死表格

本文档不是数据库字段清单，也不是 UI 截图草图，而是给 Codex / 后端 / 前端 / 数据维护侧共同遵守的产品与实现口径。

## 二、当前基线

### 2.1 当前已经存在的能力

当前装备库第一版已经完成：

- 装备分类入口
- 装备列表页
- 装备详情页
- 后端分页 / 筛选 / 搜索
- 详情页主型号 + 子型号（variants）展示
- 装备关联帖子
- 装备详情页跳转求推荐
- Excel -> PostgreSQL -> searchData/Data.js 的维护链路

### 2.2 当前已拍板的数据结构基础

当前 gear 数据底座继续沿用：

- `gear_brands`
- `gear_master`
- `gear_variants`

其中：

- `gear_master` 承接主型号级信息
- `gear_variants` 承接子型号/具体规格
- 非标准化规格继续允许通过 `raw_json` 保留
- `searchData/Data.js` 继续作为本地搜索辅助索引

### 2.3 当前页面角色

- `pkgGear/pages/index`：分类入口
- `pkgGear/pages/list`：列表与筛选核心页
- `pkgGear/pages/detail`：详情展示页

### 2.4 当前约束

本方案必须遵守：

1. 不推翻当前 `GET /mini/gear/brands` / `GET /mini/gear/list` / `GET /mini/gear/detail` 主链路
2. 不要求一次性把装备库重做成大而全导购系统
3. 不把对比页做成“自动告诉用户买哪个”的算法页
4. 不要求第一版就解决全部装备的人工理解字段
5. 继续允许 Excel 作为长期维护源
6. 继续兼容不同装备类型 detail 字段结构不同的现实
7. 装备库和求推荐的联动，优先做“预填跳转”，不做自动结论

## 三、产品北极星

GearSage 装备库不是电商货架，也不是参数百科。

它真正要承担的，是三层价值：

### 3.1 第一层：查得到
用户能快速找到想看的那类装备。

### 3.2 第二层：看得清
用户能把参数、子型号差异、关键信息看清，而不是在官网图文和淘宝参数表里来回跳。

### 3.3 第三层：比得明白
当用户已经缩小到“同类、同用途、同阶段”的候选对象时，GearSage 能帮助他把差异看清，并在看完还纠结时顺滑进入求推荐。

一句话收束：

> GearSage 装备库的任务不是替用户直接下购买结论，而是让用户更高效地完成“查找 -> 看懂 -> 比较 -> 继续求助”的过程。

## 四、当前 UI 真正缺的不是“更多参数”，而是“信息分层”

当前装备库的问题，不在“能不能展示很多参数”，而在“什么信息该在什么层被看见”。

所以产品结构必须从“纯详情表”升级为以下四层。

### 4.1 层 1：识别层
回答“这到底是什么装备”。

包括：

- 品牌
- 主型号
- 图片
- 大类
- 子类
- 年份
- 关键别名/俗称
- 一句话定位（可后续补）

### 4.2 层 2：官方规格层
回答“厂家明确给了什么客观参数”。

包括：

- 长度 / 重量 / 速比 / 容线 / 卸力 / 饵重 / 调性 / 强度 / 潜深等
- 这些字段以官网和规范化数据源为准
- 不应该混进太多编辑理解

### 4.3 层 3：GearSage 解释层
回答“这些参数意味着什么”。

包括：

- 参数旁解释
- 关键差异提示
- 同一主型号不同子型号的主要区别
- 对新手来说哪些参数值得看，哪些可以先忽略

这是当前 UI 最缺的层。

### 4.4 层 4：决策承接层
回答“看完还纠结怎么办”。

包括：

- 关联帖子
- 去求推荐
- 对比页继续求推荐
- 未来可能的“常见纠结点”专题入口

## 五、装备库信息架构总图

### 5.1 首页（分类入口）
建议保持简洁：

- 鱼竿
- 渔轮
- 鱼饵
- 未来：鱼线 / 鱼钩 / 其他

当前不建议在首页加入太多推荐逻辑。

首页目标只是：
- 快速进入大类
- 不制造决策压力

---

### 5.2 列表页
列表页目标是：

> 快速缩小候选范围。

当前已有分页、筛选、搜索基线，可以继续沿用，但 UI 上建议补强三件事：

#### A. 当前筛选上下文条
显示当前用户已经加了哪些条件。

例如：
- 水滴轮
- 品牌：Shimano
- 类别：BFS

#### B. 列表卡的“可比较性提示”
不是只展示名字和图，建议最少补一行关键信息。

不同类型不同：

- 渔轮：类别 / 速比范围 / 自重范围 / 线杯类型（如有）
- 鱼竿：长度范围 / 强度 / 调性 / 饵重范围
- 鱼饵：类型 / 长度 / 重量 / 水层 / 动作

#### C. 加入对比入口
列表页应该允许用户把候选项放入比较池，而不是必须先点进详情再返回。

---

### 5.3 详情页
详情页当前不应再只是：

- 图片
- 名称
- 一张长长的参数表
- 关联帖子

而应改成以下结构：

#### 1. 顶部识别区
- 主图
- 品牌
- 主型号
- 一句话定位（后续可补）
- 加入对比
- 去求推荐

#### 2. 子型号选择区
这是详情页核心，不要藏。

要求：
- 明确列出该主型号下所有子型号
- 让用户先选子型号，再看参数
- 子型号切换必须快，不要让用户来回跳页

#### 3. 核心参数卡
只展示最重要、最常被用户看的一层参数，不要一开始扔一整张大表。

#### 4. 详细规格表
完整展示官方规格。

#### 5. GearSage 补充说明区
放那些不一定来自官网，但对理解很重要的内容，例如：
- 该系列主要差异是长度+硬度+调性组合
- 该系列子型号主要差异是速比+线杯深浅
- 某字段对新手通常不必过度在意

#### 6. 关联内容区
- 关联帖子
- 去求推荐

---

### 5.4 对比页
对比页应该成为装备库最值钱的页面之一。

但前提是：

> 它只负责把差异看清，不负责直接替用户给结论。

所以对比页不是“推荐页”，而是“差异页”。

---

## 六、对比的总原则

### 6.1 只允许同类型比较
第一版严格限制：

- 水滴轮 vs 水滴轮
- 纺车轮 vs 纺车轮
- 枪柄竿 vs 枪柄竿
- 直柄竿 vs 直柄竿
- Vib vs Vib
- Minnow vs Minnow

不建议放开跨类型自由比较。

因为跨类型自由比较只会制造噪音。

---

### 6.2 比较对象是“子型号”，不是主型号大而泛的抽象比较
真正被买的是子型号。
用户纠结的也通常不是一个系列，而是：

- C3000 和 4000
- 6'8 ML Fast 和 7'0 M Fast
- 110SP 和 130SP

所以对比页应该以 `variant` 为核心，而不是只比 `master`。

---

### 6.3 对比默认假设：用户已经知道自己在比较同作用装备
也就是说：

- 已经缩小到了同用途候选
- 不是在拿溪流竿和岸投长竿乱比
- 不是在拿 VIB 和米诺乱比

这会直接影响页面展示逻辑：

#### 默认状态
不反复展示那些已经相同的“用途标签”。

例如：
- 同为泛用纺车轮，不要满屏都显示“泛用：是”
- 同为溪流短竿，不要把“溪流”做成每列重复主信息

#### 例外状态
如果系统发现候选项之间出现明显用途冲突，再提示：

- 这几项的使用方向不完全一致
- 可能是“精细 vs 远投”
- 可能是“泛用 vs 障碍”
- 建议先确认自己是否对比错对象

也就是：

> 用“差异警告”代替“无差别重复展示”。

---

### 6.4 对比页要区分“官方参数”和“GearSage 归纳”
这件事非常重要。

#### 官方参数
- 来自官网、Excel、结构化源
- 可以直接比

#### GearSage 归纳
- 泛用 / 精细 / 障碍 / BFS / 岸投 / 溪流
- 适合轻饵 / 适合重饵
- 多少克起抛
- 是否偏新手友好
- 这些都不是严格官网参数

所以 GearSage 归纳必须：

1. 单独分区
2. 明确标为“GearSage 归纳”
3. 不要伪装成厂家官方事实
4. 支持后续慢慢补，不要求一次性全覆盖

---

## 七、对比深度分层模型

不同装备类型，应该允许不同深度。

我建议统一做一个四层模型。

### L0：快速对比
用于列表页或详情页底部小卡片。

内容少，只用于把候选拉起来。

### L1：核心参数对比
对比页默认展开层。

展示用户最关心的 5~8 个字段。

### L2：完整规格对比
展开后展示所有官方规格。

### L3：GearSage 解释层
展示人工归纳、使用提示、错误对比提醒、求推荐承接。

这样做的好处是：
- 不会一上来压死用户
- 同时保留深度
- 不同装备类型可以在 L1 与 L3 里有不同重点

## 八、渔轮对比设计

## 8.1 渔轮对比先分大类

先分：

- 纺车轮
- 水滴轮

这两类的对比重点不同，不能混一套。

---

## 8.2 纺车轮对比逻辑

### 纺车轮的真实子型号差异
通常主要来自：

- 深浅线杯
- 速比
- 尺寸
- 也可能存在摇把长度或细微版本差异

### 纺车轮对比时，L1 核心字段建议
默认展示：

1. 价格
2. 子型号名称
3. 速比
4. 自重
5. 最大卸力
6. 容线量（按最常用标准展示一组）
7. 线杯类型（浅 / 中 / 深，若能归纳）
8. 协力报警：不显示
   - 因为纺车轮默认都有，这属于无效差异

### L2 完整规格建议
包括：
- 多线径容线数据
- 轴承数
- 摇把长度（如果有）
- 收线量 / 每圈收线
- 官方技术名词列表（只列，不做强比）

### L3 GearSage 归纳建议
可后续逐步补：

- 更偏轻饵 / 更偏泛用 / 更偏远投
- 更适合细线
- 更适合持续抛投
- 更偏保守耐用 or 更偏轻量激进

注意：
- 这些不是第一版强依赖字段
- 覆盖不足时宁可不显示，也不要瞎补

---

## 8.3 水滴轮对比逻辑

### 水滴轮的真实子型号差异
通常主要来自：

- 速比
- 深浅线杯（少量）
- 左右手
- 部分系列会有 70 / 100 / 150 尺寸差

### 左右手处理原则
左右手在购买中重要，但在对比逻辑中不应成为“实际性能差异字段”。

所以：

- 详情页仍可分左右手展示
- 对比页不应允许“同一款左手 vs 右手”作为有效比较
- 如果用户加进来了，应自动合并或提醒无意义比较

### 水滴轮 L1 核心字段建议
默认展示：

1. 价格
2. 子型号名称
3. 速比
4. 自重
5. 最大卸力
6. 容线量
7. 刹车系统
8. 协力报警（是否有）
9. 线杯深浅（若能确定）
10. 最低适投克重（仅 GearSage 归纳，非官方层）

### 关于“多少克起抛”
这是非常典型的 GearSage 解释层字段，不是官方对比层字段。

建议处理方式：

- 不放在官方参数区
- 放在 GearSage 归纳区
- 文案明确：`GearSage 归纳 / 非官方参数`
- 支持以下值：
  - `约 3g 起`
  - `约 5g 起`
  - `更偏中轻饵`
  - `更偏泛用`
- 如果没有足够把握，不显示

### 关于刹车系统
刹车系统非常重要，但品牌命名混乱。

建议做双层展示：

- 官方技术名：原样展示
- GearSage 归一标签：磁力 / 离心 / 电子 / 混合 / 品牌自定义待确认

用户真正需要看的，是归一标签。  
高级用户可以点开看原始技术名。

## 九、鱼竿对比设计

## 9.1 鱼竿先分：
- 直柄竿
- 枪柄竿

因为这本身就决定了搭配、使用方式和用户心智。

---

## 9.2 鱼竿子型号的真实差异
通常主要来自：

- 长度
- 硬度（L / ML / M / MH 等）
- 调性（F / MF / XF 等）
- 饵重范围
- 节数
- 竿稍结构
- 适线范围

主型号下子型号非常多。  
所以鱼竿详情页和对比页都要围绕“选子型号”来设计，而不是围绕“品牌故事”。

---

## 9.3 鱼竿 L1 核心字段建议
默认展示：

1. 价格
2. 子型号名称
3. 长度
4. 强度
5. 调性
6. 饵重范围
7. 自重
8. 节数
9. 收纳长度
10. 最大适线值 / 适线范围

这是最典型的“买前马上会比”的字段。

---

## 9.4 鱼竿 L2 完整规格建议
包括：
- 材质
- 导环信息
- 先径 / 元径
- 产地
- 官方技术名
- 更详细线号范围
- 是否实心竿稍（如果有）
- 竿把形式
- 其他细节字段

---

## 9.5 鱼竿 L3 GearSage 归纳建议
“泛用 / 精细 / 障碍 / 岸投 / 溪流”这些并不是官网参数，但对用户很重要。

所以建议这样做：

### 原则 1：这类字段只在“有差异”时显示
例如三根竿都被归纳为“泛用”，默认不放到核心对比区。

### 原则 2：当候选项方向冲突时，做“用途提醒”
例如：
- A：更偏精细
- B：更偏泛用
- C：更偏远投

这时页面顶部出现提醒：

`这几支竿的主方向不完全一致，可能不是同一类用途对比。`

### 原则 3：用途标签不是唯一结论，而是辅助判断
建议字段可以叫：
- `fit_style_tags`
- `gsc_usage_tags`

例如：
- 泛用
- 精细
- 障碍
- 远投
- 小饵
- 重饵
- 溪流
- 岸投

这些字段不要全铺开。  
只在：
- 有差异
- 或需要纠错提醒
时出现。

## 十、鱼饵设计

第一版先不做鱼饵对比，先把鱼饵详情展示和筛选做好，性价比更高。

原因：

1. 鱼饵颜色过多
2. 同系列子型号维度太碎
3. 实际购买判断受颜色、尺寸、潜深、季节、鱼情影响太大
4. 第一版硬做对比，价值不如把详情和筛选做到位

所以鱼饵第一版建议：

### 10.1 做好详情页
至少展示：
- 图片
- 名称
- 类型
- 长度
- 重量
- 潜深
- 浮力状态
- 钩号（如有）
- 颜色列表（如要做）
- 关联帖子
- 去求推荐

### 10.2 做好筛选
鱼饵更适合做：
- 类型筛选
- 水层筛选
- 主要动作筛选
- 推荐词 / 家族词筛选

### 10.3 暂不做正式对比页
最多保留“加入对比”按钮隐藏不开放，或者只做 very light 的参数并排，不作为主功能宣传。

## 十一、UI 设计重点补充

## 11.1 列表卡应该补什么
当前列表卡最该补的是：

### 渔轮卡
- 品牌 + 型号
- 类别
- 速比范围 / 代表速比
- 自重范围
- 价格区间（如有）
- 支持加入对比

### 鱼竿卡
- 品牌 + 型号
- 直柄 / 枪柄
- 长度范围
- 强度 / 调性范围
- 饵重范围
- 支持加入对比

### 鱼饵卡
- 品牌 + 型号
- 类型
- 长度 / 重量
- 水层 / 动作
- 先不强调加入对比

---

## 11.2 详情页应该补什么
除了已有的参数表，重点补：

1. 子型号切换区
2. 核心参数卡
3. GearSage 解释层
4. 去求推荐 CTA
5. 加入对比 CTA

---

## 11.3 对比页应该补什么
最重要的不是更多字段，而是这 5 个点：

1. 顶部固定显示候选项标题
2. 相同字段默认折叠或弱化
3. 差异字段高亮
4. 字段按类别分组
5. 底部固定“带着这几个候选去求推荐”

## 十二、字段分组建议

无论渔轮还是鱼竿，对比页都建议按组展示，不要平铺。

### A. 基础识别
- 品牌
- 子型号
- 价格
- 图片

### B. 核心性能
渔轮：
- 速比
- 卸力
- 自重
- 容线
- 刹车系统

鱼竿：
- 长度
- 强度
- 调性
- 饵重
- 自重

### C. 结构规格
渔轮：
- 线杯深浅
- 每圈收线
- 摇把长度

鱼竿：
- 节数
- 收纳长度
- 实心竿稍
- 最大线值

### D. GearSage 归纳
- 使用方向
- 是否更偏精细/泛用/远投
- 适配提醒
- 错误对比提醒

### E. 关联动作
- 查看详情
- 查看关联帖子
- 去求推荐

## 十三、数据设计补充建议

为了让后面可扩充，建议在 gear 体系里补一层“可归一、可人工维护、非官方”的扩展字段。

### 13.1 `official_specs`
- 纯官方结构化参数
- 可直接比较
- 由 Excel / 官网采集而来

### 13.2 `gsc_traits`
- GearSage 人工归纳标签
- 非官方
- 可为空
- 可渐进补充

例如：
```json
{
  "fitStyleTags": ["泛用", "远投"],
  "lineCupDepth": "shallow",
  "brakeTypeNormalized": "magnetic",
  "minLureWeightHint": "3g+",
  "solidTip": true,
  "compareWarnings": ["更偏精细"]
}
```

### 13.3 `compare_profile`
- 对比页专用整理字段
- 不是必须独立存表
- 可以由后端聚合生成

目的：
- 避免前端每次硬拼逻辑
- 也避免把 `raw_json` 直接甩给前端到处判断

## 十四、装备库与求推荐的关系

仍然沿用当前方向：

- 装备详情页 -> 去求推荐
- 对比页 -> 带着候选项去求推荐

但原则不变：

1. 不做算法推荐
2. 不伪装成自动决策
3. 只承接真实纠结

建议按钮文案继续保持克制：

- 看完还是纠结？去求推荐
- 带着这几个候选去问问别人
- 我这种情况适合吗？

## 十五、分阶段实施建议

## 第一刀：详情页重组
目标：
- 不是重做视觉，而是把“长参数表”改成“识别区 + 子型号 + 核心参数 + 详细表 + CTA”

优先级：最高

---

## 第二刀：对比能力 v1（只做渔轮 / 鱼竿）
目标：
- 支持 2~3 个子型号加入对比
- 只允许同类型比较
- 做 L1 + L2 两层
- 默认支持：
  - 纺车轮
  - 水滴轮
  - 直柄竿
  - 枪柄竿

优先级：最高

---

## 第三刀：GearSage 归纳字段体系
目标：
- 不要求一次性补全
- 先补最值钱、最稳定的归一字段：
  - 刹车系统归一
  - 线杯深浅归一
  - 是否实心竿稍
  - 用途方向标签
  - 错误对比提醒

优先级：中高

---

## 第四刀：鱼饵详情增强
目标：
- 先把鱼饵详情和筛选做强
- 对比暂缓

优先级：中

---

## 第五刀：未来再考虑“常见纠结专题 / 对比后求推荐”
优先级：后置

## 十六、当前明确不做

第一版不做：

1. 跨类型比较
2. 鱼饵正式对比页
3. 自动推荐最佳购买结论

5. 把所有 GearSage 归纳字段都做全
6. 用人工标签冒充官方事实
7. 泛社区式“大家都在比什么”热榜
