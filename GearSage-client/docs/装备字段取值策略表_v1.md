## 一、文档目的

这份文档用于明确：

1. 哪些字段只能来自官网
2. 哪些字段允许用白名单辅助站补充
3. 哪些字段需要两个以上来源交叉确认
4. 哪些字段不应自动写入，必须人工确认
5. 哪些字段属于 GearSage 归一层，只能 derive，不能冒充官网事实

---

## 二、统一原则

### 2.1 来源优先级

默认优先级：

1. 品牌官网 / 官方页面
2. 白名单辅助站
3. 人工确认
4. GearSage derive

默认禁止：

- 非白名单站点直接写入最终字段
- 将玩家主观看法冒充为官方结构字段
- 单一弱来源自动写入高风险字段

### 2.2 取值策略枚举

| strategy_key | 含义 |
|---|---|
| `official_only` | 只接受官网/官方页面 |
| `official_or_whitelist` | 官网优先；官网缺失时允许白名单辅助站补 |
| `multi_source_required` | 至少两个白名单来源一致才可写入 |
| `derive_only` | 不直接抓取写值，只允许 GearSage 规则归一 |
| `manual_only` | 不自动写入，必须人工确认 |

### 2.3 证据要求

| evidence_level | 含义 |
|---|---|
| `exact_text` | 页面中能找到明确字段原文 |
| `structured_table` | 页面表格/参数区直接给出 |
| `cross_source_match` | 多个来源值一致 |
| `manual_confirmed` | 人工审核确认 |

### 4.2.1 `gear_material` 高成本特殊字段

`gear_material` 不再按普通 `official_or_whitelist` 字段理解，而是正式收口为：**高成本特殊字段**。

原因：

- 页面里经常会出现 `Micro module gear`、`Hyper-tensile gear` 这类技术词
- 这些词不是材质
- 如果只按“有没有抓到”处理，很容易把技术词、营销词或平台印象误写成事实字段

当前 `gear_material` 只允许按三档规则处理：

#### A. `direct_write`

只有在同时满足以下条件时，才允许直接写入：

- 年款对齐
- 白名单站直接写出明确材质词
- 不接受技术词、营销词、泛化描述

这类结果当前优先进入：

- `player`

除非未来是官方源直接给值，才考虑进入 `official`。

#### B. `cross_source_inferred`

只有在同时满足以下条件时，才允许写入推断值：

- 两个独立高可信来源能组成强推断
- 至少一个来源给出明确材质词
- 另一个来源提供年款/平台/齿组关系，能支持推断

这类结果：

- 不能进 `official`
- 当前应进入：
  - `inferred`
  - 或在中间层标记为 `player/inferred`

#### C. `manual_required`

出现以下任一情况，必须留空：

- 找不到明确材质词
- 只有技术词或营销词
- 只有视频/论坛转述
- 年款无法对齐
- 只有单一弱来源

补充约束：

- 视频/论坛最多只能作为线索
- 不能进入 `direct_write`
- 不能进入 `cross_source_inferred` 的正式证据链

## 五、白名单站使用规则

### 5.1 可接受站点必须满足

- 有稳定 URL 结构
- 有明确品牌/型号对应关系
- 字段不是纯转载拼接
- 页面信息能追溯

### 5.2 白名单站点角色

| role | 含义 |
|---|---|
| `official` | 官方站 / 官方地区站 |
| `dealer_structured` | 经销商参数站，参数结构较清晰 |
| `reference_catalog` | 资料库/目录站，可作辅助参考 |
| `community_reference` | 玩家社区，仅供线索，不直接落最终字段 |

### 5.3 默认规则

- `community_reference` 不直接写最终结构字段
- `reference_catalog` 只可用于低风险结构字段
- 改装字段默认不能只靠单站写入

### 9.0 年款约束

新增一条硬约束：

- 同系列不同年份，`body_material`、`gear_material` 可能不同
- 所以后续不管是 experiment、review、patch，还是未来 apply，都不能只按 `model` 做模糊匹配
- 至少要优先绑定：
  - `reel_id`
  - `model_year`
  - `SKU` 或可确认的年款版本信息

当前 Shimano baitcasting reel whitelist patch 的绑定键已经收口到：

- `detail_id`
- `reel_id`
- `model_year`
- `SKU`

其中 `detail_id` 是最直接的 patch 目标键，`model_year` 与 `year_scope_note` 用来提醒“同系列不同年款不要串写”。

补充约束：

- `model_year` 最好从抓取中直接填写
- 当来源页或来源 URL 能确认年份时，优先写入 `model_year`
- 对同系列产品的版本确认，优先使用 `model_year + SKU`

新增流程要求：

- `model_year` 优先由 `identity enrichment` 从白名单辅助站补齐
- 后续 whitelist 字段补值必须先读取 `identity patch`
- 不再把 `model_year` 缺失当成人工 review blocker

### 9.2.1 body_material 评审口径

- `approved_value`：
