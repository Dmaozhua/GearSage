## 一、文档目的

这份文档用于明确：

1. 哪些字段只能来自官网
2. 哪些字段允许用白名单辅助站补充
3. 哪些字段需要两个以上来源交叉确认
4. 哪些字段不应自动写入，必须人工确认
5. 哪些字段属于 GearSage 归一层，只能 derive，不能冒充官网事实

---

## 二、统一原则

### 2.1 来源优先级

默认优先级：

1. 品牌官网 / 官方页面
2. 白名单辅助站
3. 人工确认
4. GearSage derive

默认禁止：

- 非白名单站点直接写入最终字段
- 将玩家主观看法冒充为官方结构字段
- 单一弱来源自动写入高风险字段

### 2.2 取值策略枚举

| strategy_key | 含义 |
|---|---|
| `official_only` | 只接受官网/官方页面 |
| `official_or_whitelist` | 官网优先；官网缺失时允许白名单辅助站补 |
| `multi_source_required` | 至少两个白名单来源一致才可写入 |
| `derive_only` | 不直接抓取写值，只允许 GearSage 规则归一 |
| `manual_only` | 不自动写入，必须人工确认 |

### 2.3 证据要求

| evidence_level | 含义 |
|---|---|
| `exact_text` | 页面中能找到明确字段原文 |
| `structured_table` | 页面表格/参数区直接给出 |
| `cross_source_match` | 多个来源值一致 |
| `manual_confirmed` | 人工审核确认 |

| field_key | 中文名 | strategy | allowed_sources | pass_condition | 备注 |

## 五、白名单站使用规则

### 5.1 可接受站点必须满足

- 有稳定 URL 结构
- 有明确品牌/型号对应关系
- 字段不是纯转载拼接
- 页面信息能追溯

### 5.2 白名单站点角色

| role | 含义 |
|---|---|
| `official` | 官方站 / 官方地区站 |
| `dealer_structured` | 经销商参数站，参数结构较清晰 |
| `reference_catalog` | 资料库/目录站，可作辅助参考 |
| `community_reference` | 玩家社区，仅供线索，不直接落最终字段 |

### 5.3 默认规则

- `community_reference` 不直接写最终结构字段
- `reference_catalog` 只可用于低风险结构字段
- 改装字段默认不能只靠单站写入

## 七、实验通过标准

第一轮实验默认通过标准：

1. 错误率尽量接近 0
2. 允许留空，不允许编造
3. 自动补值必须能回溯来源
4. 同一字段前后口径一致
5. 审核后才扩大到更多字段/更多品类
