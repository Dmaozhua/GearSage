## 一、文档目的

这份文档用于明确：

1. 哪些字段只能来自官网
2. 哪些字段允许用白名单辅助站补充
3. 哪些字段需要两个以上来源交叉确认
4. 哪些字段不应自动写入，必须人工确认
5. 哪些字段属于 GearSage 归一层，只能 derive，不能冒充官网事实

---

## 二、统一原则

### 2.1 来源优先级

默认优先级：

1. 品牌官网 / 官方页面
2. 白名单辅助站
3. 人工确认
4. GearSage derive

默认禁止：

- 非白名单站点直接写入最终字段
- 将玩家主观看法冒充为官方结构字段
- 单一弱来源自动写入高风险字段

### 2.2 取值策略枚举

| strategy_key | 含义 |
|---|---|
| `official_only` | 只接受官网/官方页面 |
| `official_or_whitelist` | 官网优先；官网缺失时允许白名单辅助站补 |
| `multi_source_required` | 至少两个白名单来源一致才可写入 |
| `derive_only` | 不直接抓取写值，只允许 GearSage 规则归一 |
| `manual_only` | 不自动写入，必须人工确认 |

### 2.3 证据要求

| evidence_level | 含义 |
|---|---|
| `exact_text` | 页面中能找到明确字段原文 |
| `structured_table` | 页面表格/参数区直接给出 |
| `cross_source_match` | 多个来源值一致 |
| `manual_confirmed` | 人工审核确认 |

## 五、白名单站使用规则

### 5.1 可接受站点必须满足

- 有稳定 URL 结构
- 有明确品牌/型号对应关系
- 字段不是纯转载拼接
- 页面信息能追溯

### 5.2 白名单站点角色

| role | 含义 |
|---|---|
| `official` | 官方站 / 官方地区站 |
| `dealer_structured` | 经销商参数站，参数结构较清晰 |
| `reference_catalog` | 资料库/目录站，可作辅助参考 |
| `community_reference` | 玩家社区，仅供线索，不直接落最终字段 |

### 5.3 默认规则

- `community_reference` 不直接写最终结构字段
- `reference_catalog` 只可用于低风险结构字段
- 改装字段默认不能只靠单站写入

## 九、导出开关与人工审核执行

### 9.0 年款约束

新增一条硬约束：

- 同系列不同年份，`body_material`、`gear_material` 可能不同
- 所以后续不管是 experiment、review、patch，还是未来 apply，都不能只按 `model` 做模糊匹配
- 至少要优先绑定：
  - `reel_id`
  - `model_year`
  - `SKU` 或可确认的年款版本信息

当前 Shimano baitcasting reel whitelist patch 的绑定键已经收口到：

- `detail_id`
- `reel_id`
- `model_year`
- `SKU`

其中 `detail_id` 是最直接的 patch 目标键，`model_year` 与 `year_scope_note` 用来提醒“同系列不同年款不要串写”。

### 9.1 whitelist enrichment 开关

`to_excel_shimano_bc.js` 新增环境变量开关：

- `GS_ENABLE_SHIMANO_BC_WHITELIST_ENRICHMENT`

规则：

- 默认关闭
- 只有当该变量显式设为 `1` 时，`to_excel_shimano_bc.js` 才会读取 whitelist experiment JSON，把人工审核后的候选值写回 import 导出链

### 9.2 人工审核怎么执行

1. 打开：
   [`2026-04-16_shimano_baitcasting_reel_whitelist_review.xlsx`](/Users/tommy/GearSage/GearSage-client/pkgGear/data_raw/experiment_reports/review/2026-04-16_shimano_baitcasting_reel_whitelist_review.xlsx)
2. 在 `review_candidates` sheet 中逐行核对：
   - `candidate_value`
   - `source_url`
   - `evidence_text`
3. 审核位填写：
   - `review_status`
   - `approved_value`
   - `reviewer`
   - `review_note`
4. `reason_code_stats` sheet 用来查看为什么其余记录没有进入候选补值
5. 只有在人工审核完成后，才允许打开 `GS_ENABLE_SHIMANO_BC_WHITELIST_ENRICHMENT=1` 再执行导出

### 9.3 本轮边界

本轮 experiment 的目标只是：

- 形成可审核候选
- 形成 reason_code 统计
- 让导出脚本具备可控开关
- 生成 reviewed workbook、approved patch 与 dry-run diff

不包含：

- 自动扩大到更多 Shimano 型号
- 自动扩大到 spinning reel
- 改最终总表
- 导入数据库
- 自动 apply patch
