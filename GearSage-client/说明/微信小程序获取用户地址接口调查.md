当前调用微信小程序wx.chooseAddress接口的逻辑规则需遵循以下要点：
一、权限与配置要求
接口权限开通
自 2022 年 7 月 14 日后发布的小程序，需在小程序管理后台的「开发」-「开发管理」-「接口设置」中自助开通该接口权限。若未开通，代码提审时将被拦截。
app.json 声明
需在app.json的requiredPrivateInfos字段中声明该接口，否则调用会失败：
json
{
  "requiredPrivateInfos": ["chooseAddress"]
}


2022 年 7 月 14 日前发布的小程序不受此限制。
隐私协议声明
需在小程序隐私协议中明确说明使用地址信息的场景，确保符合用户隐私保护规范。
二、调用条件与流程
使用场景限制
仅对具备地理位置强相关场景的小程序开放（如电商购物、物流配送），其他场景可能无法通过审核。
用户授权流程
首次调用：直接调用wx.chooseAddress会触发授权弹窗，用户同意后返回地址信息。
拒绝授权后：需引导用户通过wx.openSetting重新授权，代码示例：
javascript
wx.chooseAddress({
  fail: (err) => {
    if (err.errMsg.includes('auth deny')) {
      wx.showModal({
        content: '需要地址权限以完成操作',
        success: (res) => {
          if (res.confirm) {
            wx.openSetting({
              success: (settingRes) => {
                if (settingRes.authSetting['scope.address']) {
                  wx.chooseAddress(); // 重新调用接口
                }
              }
            });
          }
        }
      });
    }
  }
});

需注意 iPhone 真机可能返回fail authorize no response错误，需在判断中兼容此情况。
三、接口返回参数
接口返回的地址信息包含以下字段：
参数名	类型	说明
userName	String	收货人姓名
postalCode	String	邮编
provinceName	String	国标第一级地址（省 / 自治区）
cityName	String	国标第二级地址（市）
countyName	String	国标第三级地址（区县）
streetName	String	国标第四级地址（街道）
detailInfo	String	详细地址信息
nationalCode	String	国家码（如510000）
telNumber	String	收货人手机号
四、错误处理与兼容性
常见错误及解决方案
fail the api need to be declared in requiredPrivateInfos：未在app.json中声明chooseAddress，需补充配置。
auth deny或fail authorize no response：用户拒绝授权，需引导至设置页重新授权。
低版本兼容：基础库低于 1.1.0 时需使用wx.canIUse检测并提供替代方案。
地址信息完整性
部分用户反馈未获取到streetName字段，需确保基础库版本不低于 1.1.0，并检查接口返回是否包含该字段。
五、隐私保护与合规
数据存储与传输
地址信息需加密存储，避免明文传输，并仅用于声明的业务场景（如订单配送）。
用户数据控制
需提供地址管理入口，允许用户随时修改或删除已保存的地址。
六、插件与基础库要求
插件支持
插件调用需小程序基础库版本≥2.16.1，并在插件配置中声明requiredPrivateInfos。
多语言支持
当前接口返回的地址信息默认仅支持中文，国际化场景需自行处理语言转换。
七、调用频率与性能优化
频率限制
官方未明确限制调用次数，但频繁调用可能触发微信风控，建议结合业务场景合理设计调用逻辑。
性能优化
首次调用前通过wx.getSetting检测授权状态，避免无效调用。
缓存已获取的地址信息，减少重复调用。
代码示例
javascript
// 调用接口
wx.chooseAddress({
  success: (res) => {
    console.log('地址信息:', res);
    // 处理地址数据
  },
  fail: (err) => {
    console.error('调用失败:', err);
    // 处理错误（如引导授权）
  }
});
