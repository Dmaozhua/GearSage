# 收货地址功能完善说明

## 功能概述

本次更新完善了edit-profile页面的收货地址模块，实现了以下核心功能：

1. **移除模拟数据**：不再使用硬编码的测试地址数据
2. **微信地址接口集成**：优先调用微信小程序获取用户地址接口
3. **手动地址创建**：当微信接口失败时，提供手动创建地址的流程
4. **地址管理功能**：支持查看、编辑、删除、设为默认等操作

## 技术实现

### 1. 权限配置

在 `app.json` 中添加了地址权限声明：

```json
{
  "requiredPrivateInfos": ["chooseAddress"]
}
```

### 2. 核心功能实现

#### 2.1 地址数据管理

- **数据存储**：使用 `wx.getStorageSync('userAddresses')` 本地存储
- **数据结构**：
  ```javascript
  {
    id: Number,           // 唯一标识
    receiverName: String, // 收货人姓名
    receiverPhone: String,// 收货人电话
    province: String,     // 省份
    city: String,         // 城市
    district: String,     // 区县
    street: String,       // 街道（可选）
    detail: String,       // 详细地址
    postalCode: String,   // 邮编（可选）
    nationalCode: String, // 国家码（可选）
    isDefault: Boolean    // 是否默认地址
  }
  ```

#### 2.2 微信地址接口集成

**调用流程**：
1. 用户点击"添加新地址"
2. 调用 `wx.chooseAddress()` 接口
3. 成功时自动转换微信地址格式并保存
4. 失败时根据错误类型进行相应处理

**错误处理**：
- `auth deny` / `authorize no response`：引导用户授权
- `requiredPrivateInfos`：提示配置错误
- 其他错误：进入手动创建流程

#### 2.3 手动地址创建

当微信接口不可用时，提供三步式地址输入：
1. 输入收货人姓名
2. 输入手机号
3. 输入完整地址（省市区+详细地址）

包含简单的地址解析功能，自动识别省市区信息。

#### 2.4 地址管理操作

- **查看地址**：列表展示所有已保存地址
- **设为默认**：点击"设为默认"按钮
- **编辑地址**：点击"编辑"按钮（预留功能）
- **删除地址**：点击"删除"按钮，带确认提示

### 3. UI/UX 优化

#### 3.1 界面状态

- **有地址时**：显示地址列表 + "添加新地址"按钮
- **无地址时**：显示"暂无收货地址"提示 + "添加新地址"按钮

#### 3.2 视觉设计

- **默认地址**：蓝色边框和背景，带"默认"标签
- **操作按钮**：不同颜色区分功能（蓝色-设为默认，绿色-编辑，红色-删除）
- **响应式交互**：按钮点击效果，加载状态提示

## 代码结构

### 主要方法

| 方法名 | 功能描述 |
|--------|----------|
| `loadAddresses()` | 从本地存储加载地址列表 |

## 使用说明

### 用户操作流程

1. **添加地址**：
   - 点击"添加新地址"按钮
   - 系统优先尝试获取微信地址
   - 如果失败，可选择手动输入

2. **管理地址**：
   - 查看：地址列表自动显示
   - 设为默认：点击"设为默认"按钮
   - 删除：点击"删除"按钮，确认后删除

### 开发者注意事项

1. **权限申请**：确保小程序已在后台开通地址接口权限
2. **隐私协议**：需在隐私协议中说明地址信息使用场景
3. **地址解析**：当前使用简化解析，生产环境建议使用专业地址解析服务
4. **数据同步**：当前仅本地存储，后续可考虑与服务器同步
