用户表 （bz_mini_user）			
字段名	类型	默认值	说明（默认值原因）
id	BIGINT(32)	自增（无默认值）	数据库自动生成唯一主键，无需手动设置
open_id	VARCHAR(128)	无默认值（非 null）	注册时必须通过微信授权获取，为空则无法创建用户（唯一索引，确保非空）
union_id	VARCHAR(128)	NULL	仅多端关联时需要，默认无值（大部分场景下用户无需提供）先留空
nick_name	VARCHAR(20)	微信授权昵称（注册时填充）	新用户首次授权时同步微信昵称，默认非空（若微信昵称为空，可设为 “钓鱼人1”）
avatar_url	VARCHAR(255)	微信默认头像 URL	新用户首次授权时同步微信头像，若未获取到则用平台默认头像（如灰色头像 URL）
bio	VARCHAR(500)	''（空字符串）	个性签名默认空，用户可自行编辑
role	TINYINT	0	默认为 “普通用户”（0），管理员 / 版主需手动设置，0 - 普通用户 / 1 - 版主 / 2 - 管理员（可扩展更多角色）
status	TINYINT	0	新用户账号默认 “正常”（0），封禁 / 注销为后续操作，0 - 正常 / 1 - 封禁（禁止发帖 / 评论） / 2 - 注销（逻辑删除）
last_auth_time	DATETIME	CURRENT_TIMESTAMP	首次登录时间与注册时间一致，后续登录时更新
create_time	DATETIME	CURRENT_TIMESTAMP	注册时自动填充当前时间（数据库默认值，无需代码干预）
post_count	INT	0	新用户未发帖，初始数量为 0
reply_count	INT	0	新用户未回帖，初始数量为 0
follow_count	INT	0	新用户未关注任何人，初始关注数为 0
fans_count	INT	0	新用户刚注册，初始粉丝数为 0
like_count	INT	0	获得点赞总数，发布所有帖子的获赞数
points	INT	0	初始积分 0，通过互动行为（发帖 / 被点赞）累积，或者成就累计，规则待定
last_points_time	DATETIME	NULL	最后一次积分变更时间（用于排序 / 展示用户活跃度）
level	INT	1	默认最低等级（1 级），基于积分自动升级（如积分满 100 升为 2 级），规则待定
ship_address	VARCHAR(20)	NULL	收货地址
