# 图片上传优化实现说明

## 1. 优化概述

本次优化针对微信小程序中的图片上传功能进行了全面升级，主要包括以下几个方面：

- **智能压缩**：根据图片大小和尺寸动态调整压缩参数
- **分片上传**：支持大文件（>5MB）的分片上传
- **并发控制**：通过上传队列管理并发上传数量
- **全局配置**：在 app.js 中集中管理图片上传相关配置
- **统一接口**：提供简单易用的接口，封装复杂的上传逻辑

## 2. 核心组件

### 2.1 imageUploadUtils.js

新增的工具类，位于 `utils/imageUploadUtils.js`，提供以下核心功能：

- `smartCompressImage`：智能压缩图片
- `uploadToCloud`：上传到云存储（支持分片上传）
- `batchUploadImages`：批量上传图片
- `chooseAndUploadImages`：选择并上传图片（一站式解决方案）
- `UploadQueue`：上传队列管理类

### 2.2 mergeFileChunks 云函数

新增的云函数，位于 `cloudfunctions/mergeFileChunks`，用于服务端合并分片文件。

## 3. 使用方法

### 3.1 基本用法

最简单的使用方式是调用 `chooseAndUploadImages` 方法：

```javascript
const imageUploadUtils = require('../../utils/imageUploadUtils');

// 选择并上传图片
async function chooseAndUpload() {
  try {
    const fileIDs = await imageUploadUtils.chooseAndUploadImages({
      count: 9,  // 最多选择9张图片
      prefix: 'posts',  // 云存储路径前缀
      showLoading: true  // 显示加载提示
    });
    
    console.log('上传成功，文件ID:', fileIDs);
    return fileIDs;
  } catch (error) {
    console.error('上传失败:', error);
    return [];
  }
}
```

### 3.2 高级用法

如果需要更精细的控制，可以分步调用相关方法：

```javascript
const imageUploadUtils = require('../../utils/imageUploadUtils');

// 1. 选择图片
const res = await wx.chooseMedia({
  count: 9,
  mediaType: ['image']
});

const tempFilePaths = res.tempFiles.map(file => file.tempFilePath);

// 2. 批量上传图片
const fileIDs = await imageUploadUtils.batchUploadImages(tempFilePaths, 'custom-folder', {
  maxConcurrent: 2,  // 最大并发数
  compressionQuality: 85  // 压缩质量
});
```

### 3.3 单张图片处理

对于头像等单张图片的处理：

```javascript
const imageUploadUtils = require('../../utils/imageUploadUtils');

// 1. 智能压缩
const imageInfo = await imageUploadUtils.getImageInfo(filePath);
const compressedPath = await imageUploadUtils.smartCompressImage(filePath, imageInfo, {
  quality: 80,  // 压缩质量
  maxWidth: 400  // 最大宽度
});

// 2. 上传到云存储
const cloudPath = imageUploadUtils.generateCloudPath('avatar');
const fileID = await imageUploadUtils.uploadToCloud(compressedPath, cloudPath);
```

## 4. 全局配置

在 `app.js` 中可以设置全局配置：

```javascript
// app.js
App({
  onLaunch: function() {
    // 初始化图片上传工具配置
    this.initImageUploadConfig();
  },
  
  initImageUploadConfig() {
    this.globalData.imageUploadConfig = {
      maxConcurrent: 3,  // 最大并发上传数
      compressionQuality: 75,  // 默认压缩质量
      maxWidth: 1200,  // 默认最大宽度
      maxHeight: 1200,  // 默认最大高度
      enableWebP: true,  // 是否启用WebP转换
      chunkSize: 4 * 1024 * 1024,  // 分片大小，4MB
      chunkConcurrent: 2  // 分片并发数
    };
  },
  
  globalData: {
    // 其他全局数据...
    imageUploadConfig: {}
  }
});
```

## 6. 注意事项

1. **云函数部署**：使用分片上传功能前，需要先部署 `mergeFileChunks` 云函数。

2. **内存管理**：处理大图片时注意内存占用，尤其是在低端设备上。

3. **降级策略**：当分片上传失败时，会自动降级为普通上传。

4. **并发限制**：默认并发数为3，可根据实际情况调整。

## 7. 实现原理

### 7.1 智能压缩

根据图片大小动态调整压缩参数：
- 大于2MB：高压缩模式（质量60%）
- 大于1MB：中等压缩模式（质量70%）
- 小于1MB：低压缩模式（质量75%）

同时根据图片尺寸进行等比例缩放，确保不超过最大宽高限制。

### 7.2 分片上传

对于大于5MB的文件：
1. 将文件分割成多个小块（默认4MB）
2. 并发上传这些小块到云存储
3. 调用云函数合并这些小块
4. 删除临时的分片文件

### 7.3 上传队列

通过 `UploadQueue` 类控制并发上传数量，避免同时发起过多请求导致的性能问题。
