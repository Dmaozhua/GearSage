# GearSage 外网发布流程

本文档用于固化 GearSage remote 正式服发布流程，目标是避免本地调试环境与外网正式服在代码、装备数据、环境变量和运行状态上不同步。

## 发布边界

- 后端本地目录：`/Users/tommy/GearSage/GearSage-api`
- 后端服务器目录：`/srv/gearsage-api`
- 装备 Excel 本地目录：`/Users/tommy/GearSage-data/rate/excel`
- 装备 Excel 服务器目录：`/srv/gear-excel`
- API 域名：`https://api.gearsage.club`
- 静态资源基址：`https://static.gearsage.club/gearsage`
- PM2 进程名：`gearsage-api`
- 服务器后端仓库跟踪：`origin/main`
- 本地后端发布方式：从主仓库 `GearSage-api` 子目录拆 subtree 后推送到 `devser/main`

本流程只处理外网发布，不改变业务接口、接口返回结构、数据库表结构、上传路径、PM2 app name、Nginx、内容审核、短信或小程序业务功能。

## 本地完整发布命令

代码和装备数据一起发布时，在本机执行：

```bash
cd /Users/tommy/GearSage

git status
git add .
git commit -m "feat: gear xxx"

git branch -D api-deploy-main 2>/dev/null || true
API_SHA=$(git subtree split --prefix=GearSage-api -b api-deploy-main)
echo "API_SHA=$API_SHA"

git push --force-with-lease devser api-deploy-main:main

rsync -avz /Users/tommy/GearSage-data/rate/excel/ \
  root@49.232.195.121:/srv/gear-excel/

ssh root@49.232.195.121 "/root/deploy-gearsage.sh $API_SHA && /root/import-gear.sh"
```

`API_SHA` 是服务器必须部署到的后端 subtree commit。`/root/deploy-gearsage.sh $API_SHA` 会校验 `origin/main` 是否等于这个 SHA，不一致会直接失败，防止服务器部署到旧代码或错误代码。

## 小程序前端发布说明

服务器发布只更新后端代码和装备数据库，不会更新微信小程序前端版本。

首页、tabBar、UI、WXML、WXSS、JS 等小程序前端变更，必须通过微信开发者工具上传代码。微信审核看到的前端版本来自微信公众平台后台中的小程序版本，不来自 `/srv/gearsage-api`，也不来自 `https://api.gearsage.club`。

如果一次需求同时包含后端接口、装备数据和小程序前端变更，需要分别完成：

- 后端：subtree 推送后执行 `/root/deploy-gearsage.sh $API_SHA`
- 装备数据：rsync Excel 后执行 `/root/import-gear.sh`
- 小程序前端：微信开发者工具上传，并在微信后台确认体验版 / 审核版

## 只发布后端代码

没有装备 Excel 更新时，只发布后端代码，不执行 `/root/import-gear.sh`。

```bash
cd /Users/tommy/GearSage

git status
git add .
git commit -m "fix: backend xxx"

git branch -D api-deploy-main 2>/dev/null || true
API_SHA=$(git subtree split --prefix=GearSage-api -b api-deploy-main)
echo "API_SHA=$API_SHA"

git push --force-with-lease devser api-deploy-main:main

ssh root@49.232.195.121 "/root/deploy-gearsage.sh $API_SHA"
```

## 只更新装备数据

只更新 `/Users/tommy/GearSage-data/rate/excel/` 中的装备 Excel，且后端导入脚本没有变化时，可以只同步装备数据并执行导入。

```bash
rsync -avz /Users/tommy/GearSage-data/rate/excel/ \
  root@49.232.195.121:/srv/gear-excel/

ssh root@49.232.195.121 "/root/import-gear.sh"
```

如果 `GearSage-api/scripts/import_gear_excel.js`、`package.json`、数据库连接环境变量或装备接口相关后端代码有改动，必须先按“只发布后端代码”流程发布后端，再执行 `/root/import-gear.sh`。

## 发布前检查

发布前必须先看 `git status`。判断标准：

- 本次要发布的代码改动必须已经 commit
- 不想发布的实验改动、临时文件、调试脚本不能混进本次 commit
- 本地 subtree 发布前，应确认 `GearSage-api` 内的后端改动就是本次要上线的内容
- 装备数据发布前，应确认 `/Users/tommy/GearSage-data/rate/excel/` 是本次要发布的 Excel 基准
- 如果本次包含小程序前端变更，应确认微信开发者工具中待上传代码与本地目标版本一致

## 服务器脚本职责

### `/root/deploy-gearsage.sh`

职责：

- 只发布 `/srv/gearsage-api` 的代码
- 校验可选的 `EXPECTED_SHA`
- 强制对齐 `origin/main`
- 安装依赖、构建、重启 `gearsage-api`
- 写入 `/srv/gearsage-api/deploy-info.json`
- 验证 `health` 和 `health/db`

该脚本会执行 `git reset --hard origin/main`。服务器上不应保留未提交的业务热修；所有正式变更必须先从本地提交并通过 subtree 推送。

### `/root/import-gear.sh`

职责：

- 只导入 `/srv/gear-excel` 中的装备 Excel 数据
- 固化 `.env` 中的 `GEAR_EXCEL_DIR=/srv/gear-excel`
- 先执行 `npm run import:gear:check`
- check 通过后才执行 `npm run import:gear`
- 验证装备列表接口和帖子列表接口

该脚本不负责拉代码、不负责构建、不负责重启 PM2。代码发布由 `/root/deploy-gearsage.sh` 完成。

## 发布后核对

远端执行完成后，至少确认：

```bash
curl -fsS https://api.gearsage.club/health
curl -fsS https://api.gearsage.club/health/db
curl -fsS "https://api.gearsage.club/mini/gear/list?type=reels&page=1&pageSize=3" -H "clientid: wx_mini"
curl -fsS "https://api.gearsage.club/mini/topic/all?limit=3" -H "clientid: wx_mini"
ssh root@49.232.195.121 "cat /srv/gearsage-api/deploy-info.json"
```

## 常见失败处理

- `EXPECTED_SHA mismatch`：本地 subtree 推送没有成功，或服务器 `origin/main` 不是刚推的提交。重新确认 `API_SHA` 和 `git ls-remote devser main`。
- `npm run build` 失败：代码没有通过构建，发布中断，不继续重启。
- `health/db` 失败：服务或数据库连接异常，发布中断。
- `import:gear:check` 失败：Excel 数据源不满足导入校验，不允许继续写库。
- 装备接口验证失败：导入后外网接口不可用，脚本失败退出，需要先看 `pm2 logs gearsage-api` 和导入输出。

## 数据同步口径

本地装备数据唯一发布源是：

```text
/Users/tommy/GearSage-data/rate/excel/
```

服务器装备导入源是：

```text
/srv/gear-excel/
```

不要再使用旧路径 `/Users/tommy/GearSage/GearSage-client/rate/excel/`。
